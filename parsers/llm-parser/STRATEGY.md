# Стратегия парсинга с LLM

## Анализ старого парсера

### Что парсер делает хорошо:
1. ✅ Извлекает структуру таблицы (заголовки, подгруппы)
2. ✅ Определяет день недели, номер пары, время
3. ✅ Парсит чет/нечет недели

### Где парсер ошибается (нужна LLM):
1. ❌ Разбор ячейки: "Предмет (Тип) Преподаватель Аудитория"
   - Проблема: разные форматы, пробелы, опечатки
   - Пример: "Математика (Лекции)проф. Иванов2-203" (нет пробелов)
   
2. ❌ Извлечение дат: "до 22.03." или "С 06.04.26."
   - Проблема: разные форматы, разные позиции в тексте
   
3. ❌ Специальные аудитории: СК «Тезуче», НОЦ "Геодезия"
   - Проблема: кавычки, сокращения

4. ❌ Множественные преподаватели
   - Пример: "доц. Калманович В.Л., доц. Архипов Е.Ю."

## Оптимальный подход

### Гибридная схема:
1. **Python парсер** - извлекает структуру:
   - Заголовки групп и подгрупп
   - День недели, номер пары, время
   - Чет/нечет неделю
   
2. **LLM** - парсит только содержимое ячеек:
   - Вход: "Математика (Лекции) проф. Иванов И.И. 2-203"
   - Выход: {subject, type, teacher, room, dates}

### Преимущества:
- ✅ Меньше токенов (отправляем только ячейки, не всю таблицу)
- ✅ Быстрее (параллельная обработка ячеек)
- ✅ Дешевле (~100 токенов на ячейку вместо 10000 на файл)
- ✅ Точнее (LLM видит только нужную информацию)

## Промпт для LLM

```
Ты парсишь ячейку расписания университета.

Формат ячейки: "Предмет (Тип) Преподаватель Аудитория"

Возможные варианты:
- "Математика (Лекции) проф. Иванов И.И. 2-203"
- "Физика (Практические)доц. Петров2-305" (без пробелов)
- "Химия (Лабораторные) до 12.04. доц. Сидоров 1-36"
- "История (Лекции) С 20.04. проф. Смирнов 3-208"
- "Физкультура (Практические) СК «Тезуче» доц. Калманович В.Л."

Верни JSON:
{
  "subject": "название предмета",
  "type": "Лекции/Практические/Лабораторные",
  "teacher": "ФИО преподавателя или null",
  "room": "аудитория или null",
  "start_date": "YYYY-MM-DD или null",
  "end_date": "YYYY-MM-DD или null"
}

Правила:
- Если год не указан, используй 2026
- Убери лишние пробелы и точки
- Если несколько преподавателей, объедини через запятую
```

## Реализация

1. Python извлекает таблицу и структуру
2. Для каждой непустой ячейки → LLM запрос
3. Собираем результаты в единый массив
4. Валидация и загрузка в БД

## Оценка стоимости

- 1 файл ≈ 50 занятий
- 1 занятие ≈ 150 токенов (100 вход + 50 выход)
- 50 занятий = 7500 токенов
- Стоимость: 7500 * $0.00000014 = $0.00105 за файл
- 220 файлов = $0.23 за все расписание
